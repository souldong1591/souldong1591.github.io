hexo配合github搭建blog简易教程

  • github
  • hexo

0 前言

从去年就开始念叨要开始写blog，期间也搞过jekyll搭配github，但是视觉上总觉的不是很好，而且搭建本地环境挺麻烦的，就这样拖沓到前些天。这两天稍微闲了下来，想起来还有hexo没试过，看朋友用这个搭建的挺漂亮的，就动手搭建了出来，搭建过程遇到了好多问题，就总结个教程留到这里。

1 hexo简介

hexo是一个基于Node.js的静态博客程序，其编译上百篇文字只需要几秒。其生成的静态网页可以直接放到GitHub Pages，GitCafe Page，BAE等平台上。其搭建过程与jekyll相比还是要简单一些的，而且主题不需要你太折腾就很漂亮了。而搭建完成后，发布文章特别简单，只需要如下几个命令即可完成一般操作：

hexo n "name" #创建一篇名字为name的文章
hexo g #生成静态网页
hexo d #部署你的网页
hexo d -g #生成并部署

挺爽的吧~
github很熟悉了，就不多介绍了。

2 环境准备

安装node.js

首先到node.js官网的下载页面下载 Windows 版本, 这里有两种版本，推荐 .msi 的安装程序，包含了 Node.js 和 NPM 两个程序，比较方便，.exe 版本仅仅包含 node.js 的部分。
一路next安装，如果使用 .msi 安装程序的话，不用配置 PATH 路径了，安装的时候已经自动帮你配好了。打开一个命令行窗口，输入node -v，输出版本号信息，说明nodejs安装成功。

安装git

推荐安装msysgit，安装后要进行一些配置，可以自己搜索一下，比较简单，不多说了。

github肯定是要有的，也可以是gitcafe，建立与你用户名对应的仓库，仓库名必须为your_user_name.github.io,添加SSH公钥当然也省不了。可以查看github的帮助。

3 安装及使用hexo

安装hexo

上面环境装好了以后，就可以安装hexo了。
执行以下命令安装hexo：
npm install hexo-cli -g

初始化

执行init命令初始化hexo到你指定的目录：
hexo init <your folder>
不过建议直接cd到你要初始化的目录下，执行hexo init进行初始化
然后在你init的目录下，执行npm install
OK，下面开始正式使用hexo搭建自己的blog

生成静态页面并启动调试

在你init的目录下，执行hexo generate生成静态页面
然后执行hexo server进行调试
在浏览器中输入http://localhost:4000,最好使用chrome，然后你就会惊喜的发现漂亮的hexo页面了。

书写文章

搭建blog就是为了写文章，那么重要的一步来了。
执行hexo new [layout] "postName"命令，生成指定名称的文章至hexo\source_posts\postName.md。
其中，layout是可选参数，默认值为post，可在scaffolds目录看到这些可选参数。
可以添加自己的layout，方法就是添加一个 .md 文件即可，同时也可以编辑现有的layout，比如post的layout默认是hexo\scaffolds\post.md

主题安装

默认的主题肯定有人不喜欢，但是不用担心，hexo有很多优质的主题供你选用。
可在hexo的主题列表Hexo Themes进行浏览选择。有很多不错的主题，例如pacman，modernist，Vno，yilia，都是很不错的主题。
安装主题很方便，只需要执行(注意要在你init的目录下执行)：
git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
其中themes后面的目录名字随便写，但是要与下面配置时一致。
修改hexo根目录下 _config.yml 里的 theme: yilia
主题有更新时，可用如下命令获取更新：

cd themes/yilia
git pull

4 常用命令的总结

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub

hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
